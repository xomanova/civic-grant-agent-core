# Contributing to Civic Grant Agent

Thank you for your interest in contributing to Civic Grant Agent! This project aims to help volunteer fire departments and EMS agencies secure funding through AI-assisted grant discovery and writing.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists in GitHub Issues
2. If not, create a new issue with:
   - Clear description of the problem or suggestion
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Your environment (OS, Python version, etc.)

### Code Contributions

1. **Fork the repository**
2. **Create a branch** for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes** following our coding standards
4. **Test your changes** thoroughly
5. **Commit your changes** with clear messages:
   ```bash
   git commit -m "Add feature: description"
   ```
6. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Create a Pull Request**

### Coding Standards

- Follow PEP 8 style guide for Python code
- Use type hints where appropriate
- Add docstrings to all functions and classes
- Keep functions focused and small
- Write descriptive variable names

### Testing

- Add tests for new features
- Ensure all existing tests pass
- Test with realistic department profiles

### Areas for Contribution

We especially welcome contributions in:

- **Additional Grant Sources**: Add more grant databases and sources
- **Improved Eligibility Logic**: Enhance the custom eligibility checker
- **Better Parsing**: Improve grant information extraction
- **UI/Web Interface**: Create a web UI for the system
- **Documentation**: Improve docs, add tutorials, create videos
- **Multilingual Support**: Add support for non-English grants
- **Analytics**: Add grant success tracking and analytics

## Community Guidelines

- Be respectful and inclusive
- Help newcomers get started
- Provide constructive feedback
- Focus on solutions, not blame

## Questions?

Open a GitHub Discussion or reach out through Issues.
